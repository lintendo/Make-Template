# 基金频道主站页 (page-40) 规格文档

## 视图说明
`page-40` 对应图 40，是全局底栏中“基金”这一核心频道的聚合枢纽页（Hub）。
作为业务入口站，其最显著的特征是囊括了大量的活动入口图标（Service Grid）与动态推荐面板。设计风格偏向明亮的白色卡片堆叠与大面积的圆角运用。

## 零资源全局入口组件设计

在此级页面的搭建中，最庞大且繁杂的开发负担来自于顶层的 `10 宫格` 导流入口卡片。根据项目规范，切图上传是不被允许的。我部署了一整套 CSS 微缩原生图标体系来接管这部分渲染：

1. **CSS 原生聚合图标库 (Native Utility Icons)**
   - **Podium (领奖台 / 基金排行)**：利用纯粹的多列 Flex 容器。`.icon-podium` 排列三个高低不平的小方块，并针对最高的红色柱石写入 `content: '1'` 的伪类伪装成冠军台。
   - **Shield (盾牌 / 养老金)**：不再使用笨重的 SVG 绘图，我使用了极其先进的 CSS 矢量裁切：`clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)` 一刀切割出一个完美的盾牌，内部注入 “养老” 文本。
   - **Planet (行星轨道 / 全球投资)**：画一个绝对定位的圆，内部再用 `::after` 画一个椭圆边框并使用 `transform: rotate(-30deg)`，瞬间生成一颗带有环形星带的宇宙行星。
   - **Piggy Bank (存钱罐 / 定投专区)**：通过主元素的圆角矩形，叠加强制绝对定位的 `::before` 伪类作耳朵和 `::after` 做猪口，只用了二十行 CSS 便硬编码出了这只可爱的红头猪。

2. **全局底层组件对接结构 (Global Layout Skeleton)**
   - 在底部体系，它展示了一套双拼吸底栏体系：上面挂着金融应用必备的走马灯（Ticker）——“市场行情”。下方搭载了极为复杂的底部导航矩阵系统（Bottom Nav Bar）。
   - 这五个基础导航全部通过组合 Tailwind 原子类进行描摹（如 `持仓` 就是简单粗暴的圆角框嵌套大号字）。从而为整个项目打通了业务全景的骨架链路。
