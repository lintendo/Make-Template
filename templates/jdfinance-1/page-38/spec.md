# 基金特色榜单页 (page-39) 规格文档

## 视图说明
`page-39` 代表了理财应用中另一个非常庞大的业务入口：排行榜/榜单系统（Fund Ranking Lists）。
此页面展示了基于收益算法排列的基金卡片聚合页，由于包含了金银铜牌这种运营系视觉元素组件，因此对列表项渲染提出了一些特殊的组件化封装要求。

## 无资源沉浸式组件设计

1. ** CSS 奖牌系统 (CSS Olympic Medals)**
   - 榜单中不可避免的涉及到 Top 3 的视觉激励：即金牌、银牌、铜牌冠标。
   - 依照我们 "Zero-Asset" 的开发准则，我拒绝引入外部的 PNG 奖标资源。
   - 实现手段：构建了 `.medal-badge` 基类。通过极小的 `18px` 圆形限制配合内陷微阴影定型。
   - **材质合成**：利用不同的多重 `radial-gradient`（径向渐变）合成出了金属反光质感：
     - 金牌 (.medal-gold): `#ffe386` 至 `#f39c12`。
     - 银牌 (.medal-silver): `#e8edf3` 至 `#9ba4b5`。
     - 铜牌 (.medal-bronze): `#f6cca0` 至 `#c87a38`。
   - **绶带合成 (.medal-ribbon)**：在这个微小的图形下面，我分别通过 `::before` / `::after` 的极小 CSS 边框三角形旋转（`rotate(20deg)`）制作出了飘带分叉效果。配合中心插入一个纯代码白星 `content: '★'`，完美达成了全原生奖牌徽标。

2. **多态推荐卡片渲染逻辑 (Array Mapping Layout)**
   - 对于排行榜来说，底层逻辑必然是后端返回有序队列数据。
   - 并没有采取繁冗费力的静态节点堆叠，我在 `index.tsx` 组件内部顶端抽离出一套 Mock 数据对象数组 `rankingData`。
   - 然后通过一次 `Array.map` 循环，根据传入的状态开关动态映射出对应的行内容（例如是否有奖牌、是否拥有激活状态的大红按钮 `isActiveBtn` 还是褪色粉红按钮）。这种设计保证了 React 体系结构的严密规范。

3. **双重溢出遮罩技术 (Double Fade-Mask Application)**
   - 页面顶部存在的两套不同层级的基于水平滚动的导航球签标签体系。
   - 在它们外层皆复用了右侧溢出遮蔽遮罩 `.fade-mask-right`，这种纯白发光体从右向左渐逝掩盖了折断的文本，极大提升了 UI 的连贯沉浸感。
