# 白条权益-新手引导弹层 (1/4 步) (page-75) 规格文档

## 视图说明
`page-75` 为整个白条权益“全新等级开启”新手引导交互的第一步（1/4 步）。
与步骤 2 和 3 所呈现的底部服务弹层结构不同，本视图聚焦于页面上半部分的新等级横幅上。在被 80% 黑透遮罩笼罩的页面中，全新的 `白条V3` 会员卡片不仅没有被变暗，反而如浮空投影一般脱离了底层 DOM 流，高亮立于光幕之上。同时，下方的镭射引导线不再向下指示，而是反向**向上**直指悬浮卡片。

## 零资源前端重构技术概览

### 1. Z-axis Layer Elevation (突破遮罩的高亮投影)
通常这种镂空高亮效果需要使用 `mix-blend-mode` 或者切出 PNG 图片再叠一层。但在这里，我运用了真正的 CSS 空间升维：
- 重塑了整个卡片：为其构建了独立的 `.elevated-card` 挂载于新建的 `z-55` 容器中。底层深色遮罩 `.dark-mask` 处于 `z-40` 级，这样一来 V3 卡牌自然悬浮于所有明暗环境之上。
- 给 `.elevated-card` 添加了强烈的多源光源：`background: radial-gradient(circle at 100% 50%, #ffeeef, #ffffff)` 及自带的高反射光圈 `box-shadow: 0 10px 40px rgba(0,0,0,0.5), inset 0 0 20px rgba(255,255,255,1)`。这赋予了本无图片的盒子实打实的卡片厚度质感。

### 2. Ruby Generation via Hue Rotation (红宝石的色彩炼金术)
由于卡片右侧的吉祥物 🦆 需要捧举一颗红蓝发光的巨大红宝石，在不引入图片素材池的同时：
- 我组合了一套 Emoji 表情层来还原。🦆 + 👑 + 👐。
- 然而系统原生没有红宝石 emoji，最近似的是蓝色的异形钻石 `💎`。针对此痛点，我使用了 `filter: hue-rotate(160deg) drop-shadow(...)`，将宝石的蓝色图谱在色彩轮上正好旋转推进到深红色相，成功炼制出了红光熠熠的三维红宝石效果，并利用 `scaleY(0.9)` 略微改变其长宽比，匹配原图。

### 3. Vertical Inversion Indicator (反重力指引器)
针对向上升腾的光线：
- 重构了 `.up-indicator-box` 组件，将先前的流光渐变 `linear-gradient` 色值反向（从底朝上进行渐变 `to top`），并将 `.glow-dot-up` 钉位在容器的绝对 `top: 0px` 处，下方的光晕拉拽尾迹在渲染过程中就自然呈现出子弹射击般向上升起的重力反转表现。
