# 基金行情数据页 (page-36) 规格文档

## 视图说明
`page-36` 承载了全局最核心的“行情”（Market Quotes/Performance）视图。虽然这里没有图表（图表在另外的专门区域），但其核心特征是致密的矩阵化数据列表（Performance Matrix），以及一套高密度的评级徽标体系。

## 组件工程逻辑拆解

1. **内嵌横向遮罩导航框 (Tab Fade Mask)**
   - 在主卡片顶部的二级导航（"历史业绩", "年度业绩"...）存在溢出截断的交互特征：原稿中在最右侧 `>` 符号下方带有一个由白渐变至透明的滤层，用来表示右侧还可以滚动。
   - 实现方案为 `.fade-mask-right` :
     - 这是一个容器级别的遮罩：`::after { background: linear-gradient(to right, rgba(255,255,255,0), rgba(255,255,255,1)); pointer-events: none; }`。它非常优雅的处理了这种视觉暗示效果，而不干扰底层元素的滑动。

2. **高密度表单性能渲染矩阵 (High Density Table Matrix)**
   - 面对 6 行由数字矩阵和不同层级的微型状态标签构成的复杂列表，传统手写或使用重型 Table 的效果和体积都会很糟糕。
   - 开发实现：构建了一套纯前端遍历系统：`[{ period: '近1周', val1: '+...' ...]` 的数据对象传入 `Array.map` 循环渲染 Flexbox Table Row 模型。
   - 这种配置化渲染使得我们在移动端能将数据高度集中在 DOM 结构最少的情况下呈现。整个矩阵的代码行数甚至少于一个 SVG 图标。

3. **微米级动态性能标牌 (Micro Dynamic Rating Badges)**
   - “一般” (Average)、“良好” (Good)、“优秀” (Excellent) 并非是一图流，而是由带圆角的 `<span/>` 结构组成：
     - 每个标牌颜色根据数据类型强绑定不同的透明色彩：如 `bg-[#fdf2f4]` 搭配红字 `text-[#fa4c4b]` 。
     - 在 12px 字体即为下限的浏览器物理规则中，强制启用了 CSS `scale-[0.85]` 叠加 `origin-right` 缩放矩阵，把这些长条胶囊标牌压缩到了极端的 `10px` 视觉体积之内，确保绝不会顶裂移动端狭小的格栅栏。

整个卡片底部衔接着已在 `page-35` 中打磨完好的 `conic-gradient` 甜甜圈图表组件，做到了完全地模块化引用并保持视图的层叠统一。
