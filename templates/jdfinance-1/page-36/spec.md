# 基金产品主视角页 (page-37) 规格文档

## 视图说明
`page-37` 是“产品详情”这个重型页面的绝对顶部（Header）区域，也是整个 `33 -> 37` 系列界面的最终拼图顶层。
它涵盖了极具冲击力的红域英雄头图（Hero Header），超大数字的收益矩阵块，以及最核心的“收益走势”折线走势图。

## 无资源沉浸式组件设计

在这个界面的实现中，重点规避了三个经常导致加载缓慢的外部依赖项。

1. **水印网底英雄头图 (Watermarked Hero Background Layer)**
   - 设计稿的深红顶部并非纯色渐变，背景中隐隐有光晕和极细的白色圆形边框交错在右上角。
   - 实现方案为在 `.hero-bg` 内部注入 `.hero-circles::before` 与 `::after` 虚伪类。通过 `radial-gradient` (径向透明渐变) 以及极度克制的 `1px solid rgba(255,255,255,0.05)` (仅仅5%不透明度)，生成了具备高端品牌质感的呼吸水印底纹。

2. **原生走势微光排查图 (Native Vector SVG Micro-Chart)**
   - 原型中具有非常复杂的股票/基金走势图（一根灰暗的指数线加一根极速上扬的红色本品收益线），并辅有水平网格标尺。
   - `CSS Grid`: 直接在图表父容器上写入 `.chart-grid-bg`。由极其精确的 `linear-gradient` 构建背景栅格（`background-size: 20% 25%` 即恰好切割成五阶横线），完全抛弃 JS 图表框架内部缓慢重绘的 `canvas` 轴线逻辑。
   - `Inline Vector`: 在网格图层上方铺设 `<svg className="absolute inset-0 ... preserveAspectRatio='none'">`。通过 `<path>` 内置硬编码的两根极简矢量坐标序列（`M0,80 Q5...`），辅之以 `stroke="#f44b4c"` 红色，不仅彻底免去了拉取 ECharts 图表，而且线条渲染具备理论上的最高锐度。

3. **微型奖牌标版 (Micro Award Ribbons)**
   - 顶部的“贵金属·第1名”黄色皇冠标同样杜绝了切图。它利用了一层包含纯文本与一个自定义拼接的 `SVG` 迷你皇冠构成的。并通过左深右浅的多层 Flex 结构硬拼出了视觉上的立体排版效果。
