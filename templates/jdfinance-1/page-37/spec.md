# 基金购买付款页 (page-38) 规格文档

## 视图说明
`page-38` 呈现的是完成挑选后的交易环节：“购买” (Purchase Checkout) 界面。
与之前充满高密度可视化数据和金融挂件的页面不同，此页面核心目标是稳定、清晰的表单输入与支付确认体验。采用了通体白色的明亮界面辅以极简的输入与校验结构。

## 零资源交易流组件重构

1. **大额输入框重构 (Typographic Input Field)**
   - “买入金额”占据了最高的视觉权重。并没有使用原生的 `<input>` 组件（因为在静态原型中它容易触发浏览器的默认聚焦及蓝色外发光行为），而是使用大尺寸字体硬编码 `¥` 符号并跟随时隐现的光标系统 `animate-pulse` 模拟出了准备输入状态，同时配上淡灰色的占位符 "0.1元起购"。下置了极其清晰的单线底框。

2. **微缩特权服务徽章 (PLUS Tier Badge)**
   - 虽然同为 PLUS 黑金徽章，但在“申购费”配置行中展示的徽章体积被进一步压缩。为避免文本断裂或模糊，使用了原生的 Flex 结构与 SVG 星图标组合，配合 `scale-90 origin-right` 进行缩放操作。从而得到了一个与原生图像质感无二但体积为零的渲染级组件。

3. **两段式底部行动栏栈 (Two-Tier Action Bar Stack)**
   - 不同于之前的单行或半悬浮，支付页的悬挂底栏（Fixed Bottom Bar）由上下两个层级构成且通过细线进行了视觉分治。
   - 上半部分：“阅读协议”模块。将红色的强制要求勾选 `◯` 单点化处理。
   - 下半部分：“合计金额与支付”模块。因为初始状态金额为 `¥0.00`，所以“立即支付”按钮严格使用了非激活态组件的底色配置，并强制了光标禁行 `cursor-not-allowed` 与灰底 `bg-[#d8d8d8]`。

页面中未引入任何额外图片资产或复杂自定义 CSS 表，全部渲染依托于标准原子类。
