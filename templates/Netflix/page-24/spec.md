# Netflix Secondary Onboarding Core (Page 24)

## Overview
Replication of the second screen array in the user installation flow (`Download and go`). The goal was to render a flawlessly fluid and interactive representation of Netflix's vector iconography replacing static PNG pipelines with dynamic DOM execution parameters, strictly matching typography block geometries and the edge-to-edge block layout paradigm.

## Technical Details

### 1. Vector Propulsion Mapping
- Constructed the entire 'Rocket-Phone' using precision `strokeWidth` SVGs combined with explicit `<line>`, `<rect>`, and quadratic curved path vectors in realtime code.
- Mapped specific `#DFA02B` hex combinations to accurately emulate the atmospheric thruster traces and detached UI particles without adding DOM asset weight.
- The most complex segment required calculating and hand-routing exact cubic Bezier anchor points (`C 55 92 55 135 62 135 C 67 135 67 125 67 110`) to create hollow retro-futuristic thruster fins that do not intersect standard polygon fill bounds.

### 2. Physical Footer Constraint Simulation 
- Netflix's `SIGN IN` array sits fully flush outside of common safe-area/margin padding arrays when interacting with iOS boundaries block sequences (unlike standard forms).
- Bypassed normal grid padding logic to snap the `<button>` to `w-full px-0`, ensuring the Red base `#E50914` contacts physical screen bounds flawlessly.

### 3. Font and Layout Synchronization
- Centralized the `Help` and `NETFLIX` logo logic to utilize dual-column flexible blanking for absolute optical center calculations in a flexed row array. 
- Integrated a pure-code representation of the iOS pagination dots where the active frame generates simulated bloom `shadow-[0_0_8px_rgba(229,9,20,0.5)]` mimicking glowing physical pixels on OLEDs.
