# Slack Profile Detail Modal

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型中的本端（Self）用户配置详细信息屏。在此场景中，由于屏幕截图中表现出了同时存在的 "Edit Profile" 以及 "Message" 双重动作，这实际上展示的是一个既可以审视自己公开信息，又可以快速跃迁回自身 DM (Direct Message 自己给自己发送备忘录) 的用户详细资料落地页。

### 2. 用户目标与关键任务

- **信息归档确认**：自检当前的 Display Name，时区（`Timezone`），以及邮箱配置情况是否与预期一致。
- **状态监控与卸载**：迅速核对自己的挂机签名（Status: `🌴 Vacationing`），并在此屏幕中借由清空按钮一键卸载此状态。
- **自我通信与编辑**：双按钮提供了平行极高的操作入口：同自己对话（Message）或拉起编辑参数表单。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面严格切割为“焦点横幅”与“信息表单”两大区块，占比接近 1:1：

1. **Top Avatar Cover 层 (焦点横幅，`h-[40vh]` 视觉感)**：
   - 彻底摒弃传统系统栏，利用 100% 满铺正方形的绝对定位头像（复用了此前制作的 Viking 画像）直接通顶。
   - 于画面四角安插元数据：顶部左侧（时间系统）、顶部右侧（信号电量）、左上侧中部（纯白色的回撤 `ArrowLeft`）、以及左下部位最醒目厚重的大字号真名刻印（`Jensen`）和紧随其后的 Slack 绿活跃圆点。
2. **Action Blocks (核心操作组)**：
   - 使用 iOS 典型的全宽度切分逻辑包裹着带有重边框的矩形圆角按钮（白色底，加盖 `#C7C7CC` 厚灰边框以强调点击物态）。
3. **Data List Items (字典序列表)**：
   - 展现为两段式文本结构 —— 深灰缩小的属性名 + 黑体大字量的参数值（例如 `Email -> huang_hai_tao@yeah.net`）。所有列表间的底部分割线全部依照苹果原生的 Inset Layout 理念：也就是仅贴着文字最左侧往右铺划，故意保留左侧空白的 Padding 通道。

### 2. 数据与内容来源

图源的选用是本次实现的一大核心重点。这里采用了前序开发中现场调用的生成物件（`page-14/viking_avatar.png`），在架构层面直接调用前序本地目录实现。并在顶部叠加了由 `bg-gradient-to-b/t` 所构筑的高低位置阴影贴膜保障白色文字层无论在底层图片如何变亮都不至于失去可读性。

## 📐 布局与结构

- **沉浸式反白穿透**：通过在首屏顶部加入 `<div className="absolute top-0 w-full flex justify-between ... z-20">` 来实现原本归属设备的系统时间、信号能够直接叠印在图像素材上。
- **CSS 原生 Inset 分隔**：字典列表底部灰线的绘制技法：在容器使用了 `pl-4` 后在内层项目处宣告 `border-b`，借由盒子模型原生截断出最左方 16px 无线框保护区，达到 1:1 iOS 排版错觉。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

- **纯白统治 (Pure Slate)**：在主图下方完全没有应用 Slack 常规的 `#F2F2F7` 背景套底，而是彻头彻尾的最高明度纯白 `bg-white` 使得邮件蓝名等极为锐利清晰。
- **联系热点着色**：`#007AFF` (邮件外链色)，与 `#2BAC76` (代表 Online 极度积极的移动端在线标记色)。
