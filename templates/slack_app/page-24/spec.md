# Slack Custom Search Environment & Third-Party Keyboard 

## 📋 概述

### 1. 核心业务与场景定位

这是一个非准系统的第三方输入介入状态（Third-Party Input Modalities View）。它呈现了用户在准备使用中文搜狗（Sogou）九宫格拼音输入法进行查询前的待机状态。它在很大程度上延续了 Page 23 的基本逻辑骨架，但大幅重写了底层输入区。

### 2. 用户目标与关键任务

- **初始状态就绪**：在未键入任何有效过滤词组的纯空搜索栏中准备输入。
- **高级语言与输入模式切换**：通过强大的键盘扩展键处理中文检索与复合词的切换环境。

## 🧩 内容规划

### 1. 信息架构与模块划分

该复合组件在纵向维度由三大主要区块拼装：

1. **Root Search Controller**：
   - 包含灰度的 Placeholder 文本 `Search`，与失去焦点/初始化预备时的取消激活按键。
   - `Navigation Segments` 发生了极其微小的逆转镜像：在 Page 23 中停驻在 Files，而在此刻它受命回归高亮 `Messages` 的基础主视图，绿白底色颠倒互溶。
2. **Search Modifiers Listing (搜索修饰列表)**：
   - 因为缺乏检索字词，该组件将原生预制的所有五项快捷宏完整陈列于前，从 `from:me` 依序而下到底端横线。内容数据级与 Page 23 保持 1:1 无损同构。
3. **Sogou T9 Input Module (原生超高保真搜狗九宫格模拟组件 - `z-40`)**：
   - 这个输入系统组件拥有惊人的复杂度，垂直方向被硬性划定出了三个独立管理的高度阶层：
   - `Sogou Action Toolbar`：包含手作的极高保真搜狗品牌 S 形切口与红点标记，以及常用的颜文字、内置系统组件快捷呼出项。
   - `Sogou Input Grid Array`：极暴力的基于 Flex 对流系统切分的不规则方块结构。左侧被切割成专署 `，。？！` 的四段等差标点链区，中部采用极具物理质感的浅下沉高阶 T9 ABC/123 排列法，右列专司操控逻辑。
   - `iOS Dictation Bottom Pad`：搜狗地毯极下端漏出的是 iOS 系统强行夺回的语音控制底层系统栏位。

## 📐 布局与结构

- **非对称的多列网格微调阵列 (Asymmetrical Multi-Column Flex Arrays)**：
   - 由于 T9 中英网格在实际工业设计中并非完美平均分布（例如空格和主按键比例失衡），所以放弃使用原生 `display: grid` 组件，转而使用更为暴力的手工微调 `Flex` 参数切分系统：
   - 左侧柱状区占用基础可用视宽的 `12.5%` 计算域；中心九格区域直接以 `flex-1` 膨胀并内化嵌套 `row > flex-1` 分食空间；右侧退格回删柱被划定为强硬的 `18%`。使得其边距与间隔视觉占比达成与真实搜狗截图完美复刻的对数结果。

- **`Tab Border Inverse` (反侧标签的隐匿逻辑)**：
   - 相对于 23，绿光跳跃至左侧激活时。去除了 `border-l` 等边界线强行干预，由 `bg-[#37A783]` 本身强大的物理体量提供边界感知，而由退化变白的右侧 Tab 的自身 `border-l-[#D1D1D6]` 充当真正的物理分割，避免了渲染双线的问题。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

- **搜狗品牌激活警告色 (Sogou Active Orange)**：
   - 提取自截图的搜索执行键 `#FB5B1B`，这并非 Slack 的系统红，而是输入法自带的强刺激高纯度橘光，极具破坏式的行动感。甚至专门搭配了比本体更暗深的边缘实体下垂阴影 `#D13E06` 创建极富年代感的工业厚重反馈感。
- **按键背靠阴影沉浸 (Physical Button Under-Shadow)**：
   - 本套组件针对搜狗方阵极度密集的阵列感，配置了专门定制过的灰色实体下底阴影 `shadow-[0_1px_0_#999BA0]`。不发散无高斯模糊，彻底的 `1px` Y轴硬切投影还原玻璃下层薄膜键盘物理反馈质感。
