# Slack Starred Items Empty State

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型中的 "Starred Items" (星标收藏) 空状态页面 (Empty State)。用于当用户刚进驻工作区尚未进行任何收藏交互动作时，提供解释和预期指引。这是一个完全静态、传达反馈和缓冲心理落差的过渡型视图。

### 2. 用户目标与关键任务

- **信息获取**：感知此时该库内不存在任何文件数据，以及通过文本学习如何使用该功能 (`Here you'll find a continuously updated list...`)。
- **退出交互**：点选左上角 `X` 叉号无门槛退出当前 Modal View。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面极致剥离复杂元素，呈简明分布：

1. **Top Nav 面板**：
   - 清爽的 `#E5E5EA` `border-b` 分割头尾。
   - `Starred Items` 作为标题居中陈列。
2. **中心占位核心区 (Empty State Viewport)**：
   - 利用 Flex 的 `items-center justify-center` 在主屏幕区域对齐。
   - 包含一枚超大号 (`size={100}`) 的星标轮廓线圈 (`text-[#D1D1D6]`) 提升视觉吸附力，建立起 "收藏" 与 "空荡" 之概念桥梁。
   - 配随紧贴的粗体警示标题 (`No starred items`) 和更进一步具体指导规则的副加文字簇进行收尾。

### 2. 数据与内容来源

纯静态文案展示硬编环境，不需要流式数据或图库的外部接入。图标复用自 `lucide-react` 内在的 `<Star>` 加以尺寸膨胀。

## 📐 布局与结构

- **纵轴空间校准**：整体包装套用了 `flex-1` 让空场景占满剩余屏幕可用高度，但在此之上利用一个负外边距 (`-mt-20`) 进行适当地位置提拉补偿（Optically Center 拉拽模式），这是一个针对无底座空旷移动端常常用到的视觉心理偏移校准。
- **单行长度折断**：为了确保与 Screenshot 中文字排版对换行符的绝对忠实还原，对于第二段副标的 HTML 直接施加了 `<br />` 并配置两翼内边距 (`px-8`) 强定文字边界。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

全面使用由 `rules/design-guide` 规定的准则：

- **空虚灰阶组合**：为了创造一个 "不急迫" 且 "未完成" 的情感投射，所有元素被抹除饱和度落入极致淡灰：
  - Icon: `#D1D1D6` 加持 `opacity-80` 以显得空旷透明。
  - Text: 全程采用 `#C7C7CC` 加 `tracking-tight` 来收束整体阅读重力。

### 2. 组件状态定义

- **不可交互视觉态**：整块中央区域无任何操作触手，确保浏览纯粹性而免除多态开发分支。
