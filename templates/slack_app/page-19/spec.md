# Slack Media Upload Drawer View

## 📋 概述

### 1. 核心业务与场景定位

当前为 Slackbot 在线频道（Channel/DM）标准沟通视窗。但此时系统的交互焦点已经从基础聊天转移到了底部推起的“多媒体上传底座”（Media Upload Drawer）。这一界面截除了原生 iOS 的输入键盘，取而代之展示了 Slack 特有的快速媒体授权浮层。

### 2. 用户目标与关键任务

- **多标签富文本反馈展示**：检阅 Slackbot 对于格式化排版语句（富文本加粗、斜体、引用）渲染规则的教学示范，明确预览行内的 Markdown 变种指令视觉态。
- **底层硬件授信调度**：遇到原生级的权限墙，请求用户授权开放本地相册（Photos），用于日后直接将硬件数据灌入渠道生态。

## 🧩 内容规划

### 1. 信息架构与模块划分

分为四大独立板块由上至下无缝贴合：

1. **Brand Identity Header**：
   - 保留标准的 Slack `#Hash` Logo，并在 Slackbot 机器人的名牌后附加绿色的心形锚点标识系统活跃与特权属性。
2. **Conversation Chat Stream (对话留档滚动区)**：
   - 通过高度逼真的 CSS，复写呈现粗体 `*...*`、斜体 `_..._` 乃至引用区块 `>` 的转译呈现状态。所有被触发转译的标签都享有独特的深粉 `bg-[#F8F9F9]` 配合 `#DF1A55` 高亮单线包裹。
3. **Control Bar Bridge (悬空操作面板)**：
   - 作为桥梁：上方为带有孤立定界符 `@` 的当前待输入靶向槽，下方则是具有绝对优先级的 “Send（蔚蓝发射按钮）” 与高亮激活的 “Image 挂载按钮（纯蓝强调激活态）”。
4. **Media Permission Sheet (媒体申请权限舱)`100% z-20 层`**：
   - 左侧切出高黑 `.bg-[#222222]` 引导切换相机与相册流。
   - 右侧中央占据极大会面展示高度定制化的 “双叠卡片日出山顶” 空白占位指导图腾。

### 2. 数据与内容来源

完全规避通过外部资源链接图引取 “相册占位浮空原画”。这里手写了一段内联 SVG 组件封装：运用 `<mask id="..."><rect...></mask>` 对图形上端进行精准切角，配合底层带有逆向 `rotate(-6deg)` 的紫色叠版，原味 1:1 再现图库获取请求弹窗内的手绘配图材质。

## 📐 布局与结构

- **纯手工 SVG Grid 系统**：
   - 弃用通用图标库粗劣拉伸构图策略，为了模拟左下角绝对真实的 3x3 瓷砖方阵“布局（Grid）”图标，通过 `<div className="grid grid-cols-3 gap-[2px]">` 与 9 只白色 `6px*6px` 的矩形碎块列阵完美拼出原生效果。
- **浮窗吸底 (Bottom Flex-Anchoring)**：
   - 整体高度分发策略依赖底层 Drawer 的写死固化体格 `h-[260px]`，并借由 `.flex-1` 高度溢出填补消息队列框体（Conversation Viewport），制造弹性的输入区域拉伸兼容所有型号 iOS。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

- **富文本终端特异色系**：使用 `#DF1A55` 处理 CodeBlock/Pre 样式内嵌字元，它非常吸睛且不同于主线大红。
- **发送链交互状态色**：由于激活相册，下方的图片夹 Icon 跃迁为了极其艳丽的 `#0B82E5` ，与此同时 Slash 命令等图标变为了灰白的冷清态（Opacity 落后）。
- **左暗右明底座面板 (Bi-Color Modality)**：底座被生生分界为左侧极致消光黑的暗区（`#222`）供状态锁定，以及主承载区的珍珠白 `#FAFAFA` 进行柔和的信息透传及 Button (`#C7C7CC` border) 托管。
