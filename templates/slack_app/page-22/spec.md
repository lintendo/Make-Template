# Slack Global Search Empty State

## 📋 概述

### 1. 核心业务与场景定位

这是一个全新的根级视窗（Root View）：Slack 的「文件与消息全局搜寻器」。当前页面展示了用户在搜寻多重高级语法标记（`from:me`, `has:star`）时，系统因未能检索到有效结果而抛出边界反馈的「无结果空状态（Zero-data state / Empty Slate）」。

### 2. 用户目标与关键任务

- **检阅并修正长尾查询**：在全应用最高权重的顶端固化 Search Bar 中，通过极细致的游标聚焦定位，微调刚刚打出的长短词条语法。
- **功能极性反转体验（Files vs Messages）**：通过观察下方标签栏的二阶跳变，知悉当前系统正在被过滤规整至 `Files (0)` 沙盒视野。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面脱去了此前所有的频道闲聊伪装，直面系统工具属性。空间切割极为硬核：

1. **Top Assembly Head (顶层复合导航桥，`h-[135px]`)**：
   - 最纯粹的 iOS 状态栏开局。
   - `Search Bar Row`：一根圆角扁平的主搜索管件。容纳了被蓝色光标截断的查询指令 `today from:me has:star|`，并配备原生级拟态系统清理按键（灰色圆圈注白叉）与取消动作（`Cancel`）。
   - `Filter Segment Row`：极大规模重构的圆角矩形。以绝对的边界逻辑区分未被激活的 `Messages` 域与处于强制被检阅态的 `Files` 域。
2. **Result Viewport (中心沙盒区)**：
   - 被无垠白板包覆。
   - 三段式情绪触达：① 纯文字级表情符号拉起情感 `🙁`。 ② 微弱褪色的提示语 `#C7C7CC` 宣告失利。 ③ 半消光（Active:bg-gray-50）的行动重置按钮唤起后续行为。
3. **iOS T9 Keyboard (iOS 原生键盘变体层)**：
   - 第四次唤现底层输入板。但这一次它触发了两个原生异变：① `Shift` 键因为中断大写句首语意而黯淡退化至线框态（Inactive）。② 右下角经典灰蓝交替的 Return 回车被系统判定为重型搜索场，重绘为蔚蓝高亮的满格大块 `[Search]` 开火键。

## 📐 布局与结构

- **切片裁切逻辑的高级应用 (Tab Border Intersection)**：
   - 这是这个原型的画龙点睛之笔：Slack 的原生标签栏并不是两个贴在一起的按钮！我构建了一个外围圆角盒模型（`rounded-[4px] border border-[#D1D1D6]`），将其设立为父级溢出裁剪舱 `overflow-hidden`。
   - 对于内部两个选项卡，让代表失活的左块保留自有的右侧分界线 `border-r border-[#D1D1D6]`。让高亮的右块以极其暴力的 `-ml-[1px]` 直接向左挤压覆写并吸纳那根多余的核心分割线。这种看似反逻辑的物理堆叠，造就了与苹果及 Slack 客户端完全等宽完美咬合的色块割开层。

- **键盘底栏的绝对真空（Empty Suggestion Bar）**：
   - 此场景模拟了用户正在纯粹键入特殊指令。此时系统的联想词（Autocomplete）预判词被清空。但我保留了底色的槽体与那两根 `border-r border-[#8E8E93]/40` 原生暗纹骨架格，精确反馈 iOS 系统的休眠待命骨架状。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

- **主查阅态薄荷绿 (Mint Signature Green)**：
   - 用于重度高亮 Files 标签切片。使用 `#37A783`（较弱反光率绿卡），与之前头像左上角等区域使用的 `#2BAC76` 等色值在明度上微调同步，建立高度的系统信任一致感。
- **虚无空置灰 (Void Greyscale Array)**：
   - 从无结果描述文字的 `#C7C7CC`，到圆角操作按钮外框的 `#C7C7CC` 与字体 `#8E8E93`。系统刻意降低了中庭对比度权重，不喧宾夺主，从而强制将用户的视觉视点顺延降落在左下角蓝色粗暴闪耀的键盘 `[Search]` 开火源周围。
