# Slack File Viewer Modal

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型中的文件/图片预览器（File Viewer Modal）。当用户在聊天流或文件中点击静态图像 (如 `Group 36.png`) 时，触发此全屏沉浸式的深色模态页。用于展示长图、设计稿细节，并提供快速针对该文件进行分享、留言或星标 (Save) 的工具流。

### 2. 用户目标与关键任务

- **图像检视**：聚焦中部的画幅载体查看设计稿（此处呈现为横屏多面板 Wireframe 演示图）。
- **文件信息捕捉**：由左上角的缩略卡片快速了解提交人 (`EJ`)、时间与文件名，并可在右上角点击 `X` 取消离开。
- **互动处理**：阅读提示词（"This file hasn't been shared."），判断该文件目前处于私密/草稿阶段。调用底部工具栏对文件进行交互动作。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面严格建立在中轴线，并使用大面积纯黑背景。包含：

1. **Header 浮游层**：
   - 使用包含透黑色底(`bg-black/80`)的面罩，置顶于安全区域内。
   - `EJ` 的 Avatar 采用手工 SVG 像素画拟合技术，还原原色块形状。
   - 随附粗细对比强烈的作者名字、递进弱化的时间锚及暗调文件名。
2. **中心载体 (Image Placeholder)**：
   - 居中文本，包含对原图展示形式的模拟实现：原图中是横置的 3 块具有线框图特征的画布面板。这里通过 `MockSlide` 组件并用嵌套超细文本与极其微缩字号的 Flex 网格绘制出 3 块极富视觉欺骗性的 "假图"。
3. **Bottom Actions 层 (Footer)**：
   - **提示带**："This file hasn't been shared" 紧贴着分隔线。
   - **一像素分割线**：采用了极暗的灰色 `#333333` 在黑色底面上形成微妙切分。
   - **操作栏**：Icon 序列—打捞消息 (`MessageSquare`), 行星标收藏 (`Star`), 分享向外 (`CornerUpRight` 作为转发符号), 更多 (`MoreHorizontal`)。

### 2. 数据与内容来源

文件数据纯静态 Mock（MockSlide 等）。Header 资料针对 `Group 36.png` 强行植入，以此精准复刻当期 Screenshot 给出的唯一视界坐标。

## 📐 布局与结构

- **沉浸黑底**：根组件强绑定 `bg-black text-white` 并拦截所有可选文字的交互 (`select-none`)。
- **中心对齐**：使用 `flex-col` 加中间层 `flex-1 flex justify-center` 将图片强行压在纵向 Y 轴的视野正心。这种布局即使在大长屏设备中也会牢抓横置图片的排版。
- **iOS 适应**：忽略对原状态栏数值文本的重绘，符合 iOS Modal Present 的常见模式——状态栏文字通常会被 Modal 本身或黑色层吞并，呈现极简风格。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

全面使用由 `rules/design-guide` 规定的暗色系准则：

- **大盘底色**：极致黑 `#000000` 压阵。
- **柔光白与信息灰**：文本在纯暗状态下不再使用 `#8E8E93` 或太深的灰色，而是抬升至 `#A1A1AA` 和 `#FAFAFA` 等明度更高但不刺眼的层级。对于操作图标采用 `opacity-90` 避免边缘锯齿毛刺过于尖锐。
- **拟真 Wireframe**：`MockSlide` 组件内大肆释放蓝色边界元素 (`border-blue-600`) 与极限缩微尺寸 (`text-[2px]`)，在宏观上达到 "这看起来像是一张真实的设计输出长图" 效果。

### 2. 组件状态定义

- **按钮态 (Icon Buttons)**：现阶段全部展示图标本身形貌而没有任何 hover 状态挂载（因移动端主要取决于触摸发光而非桌面端悬停）。
- **分享态**：当前被认定为 "未分享"，因此没有评论气泡堆积的后续展开框体系。
