# Slack Workspaces Drawer Navigation

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型中的工作区漫游矩阵层（Workspaces Drawer）。当前端视图从右向左进行滑动时，底层应用会被附带高黑透遮罩的深色面板覆盖压制，暴露出深藏背后的服务器（Workspaces）罗列清单页。此页面是多团队用户高频使用的绝对核心调度中枢。

### 2. 用户目标与关键任务

- **层级穿梭确认**：审查当前所处的团队坐标，目前落点为 `#UI8` 团队。
- **并行进度感知**：感知主应用状态与边缘应用的动态情况（红点与数字角标），通过左侧切边面板实现跨团队消息红点的聚合扫描。
- **横向团队拓展**：触碰深灰色的 `+` 号容器进行全新企业微信/Slack的平行增设。

## 🧩 内容规划

### 1. 信息架构与模块划分

结构极为立体，整体可以抽象理解为三层三维板构形：

1. **Underlying Base Layer (被遮挡底层主视图 `z-0`)**：
   - 这是唯一能看到边缘被挤出的视图（约占 17% 的右侧可视区）。暴露了主页的信息通道 `Fee...`, `Ho...` 断层截断以及此前制作的维京人画像等常规数据，用以证明物理层级纵深感。
2. **Workspaces Nav Drawer (抽屉核心层 `z-20`)**：
   - 这是本体，采用了 Slack 标志性极强的茄色（Aubergine）色域——`bg-[#483949]` 底调为主控色，以更暗一度的 `#3F2E40` 作为表头区分区域。
3. **Global Fixed Elements (全局悬浮覆盖层 `z-40`)**：
   - 原生的 iOS Status Bar 包含电量与信号指示被故意强置于最上层，因为此类系统标示并不涉入应用页面的任何空间阻挡动画中，永远游离在所有卡片之上。

### 2. 数据与内容来源

图文中具有极其挑战的复合节点：UI8 服务器大楼复合徽章系统。不仅底层嵌套了带裁切特性的 `Hexagon`（六边形图案内贴 UI 字符），其右上角还堆叠了两层重度警告（纯白圆点新消息锚记 + 强蓝色 `bg-[#007AFF]` 的 `@` 提醒锚记 8），我们全部使用纯 CSS 逻辑外加边框挤压特性完美组合装裱而出。

## 📐 布局与结构

- **Z-Index 魔法与抽屉遮罩术**：通过分配宽幅：设定上浮面板控制权截死在 `w-[83%]`，并使其挂载 `.drawer-shadow` 产生极其强力的多象限右置发散型长距离黑边阴影（`shadow: 15px 0 40px`）来压制下方。底层应用（Base Layer）则是采用 `w-full px-[83%]` 保障渲染完整，这造成了浑然天成的切图透视感。
- **多边形图腾注入**：使用 `clipPath: 'polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)'` CSS 手工锻造，将普通的方形徽章变身为极其赛博风格的六边形截面底座以支持 UI8 等异构图形。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

- **特种色标**：大面积引入 `#483949` 专色（Slack Aubergine Base Dark），配合更次一级的交互圆角容器底色 `#3B2D3C`。白条侧边贴边标红 `bg-white` 高亮了目前的游标寻址位置。
- **毛玻璃胶囊（Frosted Capsule）**：底部的分页节点应用了 `bg-black/15` （半透底壳结合纯白透射指示）构建了完美的系统感。
