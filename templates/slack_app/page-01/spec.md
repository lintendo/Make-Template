# Slack Self Memo Page

## 📋 概述

### 1. 核心业务与场景定位

这是一个 Slack APP 原型的 "Self Memo" (自我备忘录/草稿箱) 页面。它模拟了 Slack 中的个人空间聊天界面，允许用户编写草稿、记录待办事项或提醒。界面当前处于键盘唤醒（输入中）状态，展示了 iOS 风格的虚拟键盘、富文本输入工具栏、历史消息列表以及原生的 Slack 顶部导航。

### 2. 用户目标与关键任务

- **信息浏览**：查看历史备忘消息，识别消息的时间和日期边界。
- **信息输入**：通过键盘输入新消息，使用下方工具栏附加对象(@提及，斜杠命令，附件，图片)。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面由上至下划分为五大主要区域：
1. **iOS 状态栏**：时间、位置服务图标、信号、4G网络、电池。
2. **Slack 顶部导航栏**：Slack 彩色 Logo、当前用户/频道名 ("huang 🌴" & "zZ" 状态)、展开菜单箭头、勿扰铃铛图标、搜索图标、更多菜单图标。
3. **内容展示区 (聊天历史)**：
   - 欢迎区域（位于顶部）：大字体的频道名 "Jensen 🌴" 及其介绍文本。
   - 消息流：按时间排序的消息，包含发送者头像（统一手绘风格）、名称、时间戳。包含未读消息分割红线 ("NEW")。
   - 新日期分割符（如 "Yesterday" 伪标签）。
4. **输入与工具栏**：
   - **交互辅助区**："Jot something down" 提示词及全屏编辑/展开图标。
   - **功能按键区**：包含 `@`、`/` (斜杠命令)、纸片/附件图标，以及图片选择器和发送按钮（"Send"）。
5. **软键盘 (iOS 标准样式)**：
   - 顶部预测联想词区栏 ("Hello", "The", ...)
   - 标准 QWERTY 九宫四行按键。
   - 底部工具行：地球图标(语言切换)、麦克风标以及 iOS Home 指示条。

### 2. 数据与内容来源

数据属于本地硬编码展示（Hardcoded Mock Data），包含预设的两条历史记录，头像为自定义的简易 SVG 手绘风格占位，时间和文案按照截图一比一预配置。

## 📐 布局与结构

- **整体布局模式**：采用 Flexbox 纵向布局 (`flex-col`, `h-screen`, 容器限制最大宽度为移动端 `max-w-md` 并居中使用 `mx-auto`)。
- **模块尺寸与限制**：
  - 聊天区通过 `flex-1` 与 `overflow-y-auto` 撑开，并支持滚动。
  - 顶部导航和底部键盘+工具栏高度固定。
  - 键盘区域使用具体的边距参数以像素级模拟原生体验 (`h-[42px]` 等键位高度)。

## 🎨 视觉规范

### 1. 主题来源与使用情况

使用项目中原有的 Tailwind CSS V4。所有的间距、颜色、字体基于通用规范，部分无法被标准 token 完全匹配的细节使用任意值 (如 `text-[13px]`, `w-[42px]`)。

### 2. 自定义细节 (Design Tokens)

- **主色/强调色**：
  - 分割线/新消息 "NEW"：#f43f5e (接近 Rose-500)。
  - Slack Logo：使用标准的红(#E01E5A)、蓝(#36C5F0)、绿(#2EB67D)、黄(#ECB22E)。
- **背景色**：
  - 软键盘背景：#d1d5db (Gray-300)。
  - 软键盘按键：#ffffff (部分功能键为 #B3B8C2)。
- **排版风格**：
  - 频道大标题：26px `font-extrabold`。
  - 消息正文：15px，黑色。
  - 时间戳：`text-xs`, 浅灰色。

### 3. 组件状态定义

当前页面为静态展示，主要交互为纯视觉反馈：键盘上的键位具有 `active:bg-gray-200` (白色按键) 或 `active:bg-white` (灰色按键) 状态以模拟触屏下的物理点击反馈。发送按钮此时暂处于未激活/禁用样式 (灰色边框，浅灰色字体，白色底)。
