# Slack Compose Mention Autocomplete

## 📋 概述

### 1. 核心业务与场景定位

这是一个至关重要的复合型输入交互切片（Complex Input Interactive Slice）。它的触发场景极速：用户在 Slackbot 会话页底部的基础文本框中直接敲击触发字符 `@`，随即拉起两套遮罩型的视图层：
1. **原生级键盘拉起 (Native T9 Keyboard Deployment)**：强行接管屏幕下三分之一。
2. **提及联想浮层下拉 (Mention Autocomplete Dropdown)**：作为自动推断拦截层弹出，切断在输入区与对话记录区之间。

### 2. 用户目标与关键任务

- **精准快速寻址**：在复杂的企业拓扑关系与系统 Bot 中迅速通过首字母点选对象。
- **状态感知**：不仅能看见名字与头像，必须在极其微小的视觉单元内同时判断该员工是活跃、离线、请假还是被强制免打扰（Snoozed）。

## 🧩 内容规划

### 1. 信息架构与模块划分

这是目前架构最立体的视图切片，垂直空间被严格切割为四层主逻辑：

1. **Top Container (`z-10`)**：
   - 包含常驻的白色顶栏 `#Slackbot` 与上半部分会话信息。在这张截图中，其内容并未变动，但由于下方容器的野蛮挤压，原先的内容被迫遮挡裁切，且 `overflow` 被强制截断。
2. **Typeahead Dropdown (`z-20`)**：
   - 一个顺势插入文档流的高保真列表。每一条记录包含 4 个强特征锚点（Status, Avatar, Bold ID, Light Name），并且支持像 `🌴 Vacationing` 这样带有行内垂直切割的高级扩展标签。
3. **Input Payload Toolbar (`z-20`, 停靠在系统键盘正上方)**：
   - 因为当前焦点是提及（Mention），操作流彻底偏转。最左侧的 `@ (AtSign)` 被瞬间染色为 `#0B82E5` 的极高亮代表处于激活域。同时，先前提及的图片库和文件系统（Paperclip）图标全数黯淡退居后台。
4. **iOS Keyboard (`z-40`, 最重权级空间占领层)**：
   - 高度还原的 iOS 系统底座键盘，相较之前的默认态，截图中处于大写锁定唤醒态（Capitalization Locked），其内部修饰键 Shift 被填充满黑墨以暗示状态。

### 2. 数据与内容来源

此页面中大量应用了复合内嵌结构创造的虚拟化“抽象高保真” Avatar 元素，脱离了单纯堆贴图的低级方案：

- 对于 `360 Mockups`、`Aakriti` 等无清晰头像的用户：我通过精细的 `<div />` 绝对位移排布切分出色彩板块，利用 `rounded-tr` 等拼装技法高强度复原了早期 Slack 的抽象生成式名牌 Logo。
- `Slackbot`：四色方块的像素拼贴极度还原系统原生质感。
- `Huang Jensen` 的 Snooze 状态指示点：纯粹利用 CSS 构建 `11*11` 空心圈并定位挂载外溢 `z` 字图层表现休眠挂起。

## 📐 布局与结构

- **极端的 Flexbox 空间算计**：
   - 我们构建的是完美的 Flex 布局，并且不使用 `absolute` 固定死 Dropdown 的上边距。我们允许 `Dropdown`, `Toolbar`, `Keyboard` 作为定高元素堆叠填充底部。
   - `ConversationFeed 背景对话层` 被设置为 `flex-1`，这就使得只要下方三大组件尺寸固定，聊天框架会自动无缝上缩挤压，制造出自然衔接被遮挡的极度真实感！完全契合流体设计的标准。
