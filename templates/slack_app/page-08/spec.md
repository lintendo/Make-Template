# Slack All Files Browser

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型中的 "All Files"（全部文件）视图。该页面聚合了工作区内出现或分享过的图像、文档。它包含两极切换体系 (`Everyone` vs `Just You`)，在当前阶段处于 "Everyone" 的浏览流下，罗列着用户上传的大量屏幕截图和图片稿件数据。

### 2. 用户目标与关键任务

- **多维筛选切换**：利用贴顶部的 Toggle 按钮快速切入 "Everyone" 全部成员产生的文件流 或 "Just You" 纯属个人的私密上传。
- **文件基本盘检视**：向下滚动阅览每个文件的简要卡片：文件原名、资源尺寸 (`281 KB`) 以及文件格式 (`PNG`)，加上精准的时间戳和来源频道的归属。
- **取消返回**：点按左上角原生的灰度 `X` 逃逸并回到前缀页面。

## 🧩 内容规划

### 1. 信息架构与模块划分

从上至下属于经典的白底列表平铺结构：

1. **Header 行**：
   - 使用 iOS 模式构建居顶部栏目。
   - 层叠正中设置有一个复合图文块 `[Layers Icon] All Files [ChevronDown]`，视觉暗示出其并非死标题而是一个可点击引发 Dropdown 抽屉（弹出过滤器或群组菜单）的核心过滤入口。
2. **两级分控选项卡 (Tabs)**：
   - 等比例对切。左侧蓝底白字高亮代表聚焦，右侧浅灰底灰字代表退让。
3. **数据清单列 (Scrollable Table)**：
   - 左侧为独具特色的缩略图示意。在这里特别模拟了带折角的文档图案（借由 Tailwind `border` tricks 刻画右上角折痕）加嵌套白色 `Image` 图形来传递出 "这是一个图像类文件" 的概念。
   - 右侧纵向布局了紧凑型 3 行信息：标题 (Darkness)、元信息 (Mid-gray)、事件时间地点信息 (Light-gray)。三级纵深对比强化阅读引导层级结构。

### 2. 数据与内容来源

文件数据由 `files` 数组 10 条真实世界记录硬编码渲染而生，涵盖了 `Screen Shot 2019-04-22 at 12.36.42 PM.png` 等具备高度截图真实感的字符案。

## 📐 布局与结构

- **列表滚动特性**：使用了外部容器包裹 `flex-1 overflow-y-auto` 并施加 `pb-[34px]` 将防误触层纳入滚动避让缓冲垫。
- **列表内部线排版**：采用横贯通底的 `.ios-border` 方案（`border-b` 控制全幅底边），每行数据行均为 `bg-white`，产生严格的账本账目观感。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

全面使用由 `rules/design-guide` 规定的准则：

- **Slack 系高光蓝**：`#1878F3` 在 Tab 选用以凸显触发点；`#5CA3E6` 用于充当轻量级文件底盘标志。
- **三重灰度梯度文本**：
  - Primary Text: `#000000` (Name)
  - Secondary Text: `#A1A1AA` (Size & Config)
  - Tertiary Text: `#C7C7CE` (Timeline base)
- **CSS Tricks 折角模拟**：文件 Icon 的右上折角是通过纯 CSS 三角形 Border 处理 `w-0 h-0 border-[10px]...` 而非引入外源 SVG 构建的，维持极客开发与极小引用的技术规范。
