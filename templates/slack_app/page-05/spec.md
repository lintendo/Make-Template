# Slack Invite People Page

## 📋 概述

### 1. 核心业务与场景定位

这是 Slack APP 原型的 "Invite People"（邀请成员）页面。处于全屏表单模态框样式，专用于输入电子邮箱去邀请新用户，或跳转第三方通讯录提取关系链。当前界面正处于“正在键入（Active Keyboard）”交互阶段，调出的是 iOS 特殊针对 `email-address` 类型定制的小写英文字母及带有 `@` 和 `.` 快捷符的软键盘。

### 2. 用户目标与关键任务

- **单独输入邀请**：在顶栏 Input 组件中通过特殊设计的 Email 键盘主动输入目标联系人的邮件地址。
- **批量导入通讯录**：点击下方的预设通道按钮直接获取系统与 Google 通讯录关系链。
- **取消返回**：点击左上角的 `X` 返回前置应用。

## 🧩 内容规划

### 1. 信息架构与模块划分

页面采用典型的 iOS Grouped Style 变种打底（`#F2F2F7`），将整个容器划分成上下多个层级：

1. **Top Nav/Header**：
   - 合并状态栏后属于 sticky 定位，操作栏核心包含 `X` 关闭按钮，粗体 `Invite People` 标题，及灰色的因未输入可用内容而呈非激活保护态的 `Send`。
2. **核心输入区**：
   - 深陷在灰色底背景中的一道独立行。带有邮件（Mail）小图标提示，以及自带浅蓝色竖线光标 (`|`) 的 Placeholder。表现由于键盘弹起获得主视线焦点。
3. **集成导流区 (Contacts Integrations)**：
   - 使用两行结构组成的白色 List Grouping 磁贴。
   - 分别指向本地联系人 (Contacts，带有带 `+` 号的绿人头图标) 以及 Google 联系人 (带有高度自拟真的黄底 ID Card 图标)。
   - 行与行间使用经典 iOS 模式：全屏起点的 `border-y` 以及自图标后起向右拉满的 `h-[0.5px]` 分割线。
4. **底部灰度短语**：居中放置文本及微笑表情，填充下半部多余视界空隙。
5. **iOS 邮件软键盘区 (Email OS Keyboard)**：
   - 该部分利用固定位置固定尺寸和 `div` 手动还原了底层。特别是在四行排布上呈现了小写字母 `font-light` 的特征。
   - Bottom Row 被切割重组，加入了 `@` 和 `.` 号快捷按键以适配输入邮箱的高频动作。

### 2. 数据与内容来源

纯静态页面。所有的交互均被表现为一种 "Active View Snapshot"（当前帧）。包含的自定义 Google Contacts ID 图标也是通过观察利用 SVG 的基础集合形状自制还原的。

## 📐 布局与结构

- 外界依赖 `h-screen max-w-md` 的容器锁定于垂直显示并保持 `flex-col` 。
- 中间用 `flex-1` 让包含着输入框、选项与笑脸的区域占满并向上堆叠空白 Spacer。
- Keyboard 组件置于最后，附带 `mt-auto z-30 relative` 以保证牢牢沉淀在屏幕容器的最底端，并提供向上拉拔的重阴影。

## 🎨 视觉规范

### 1. 自定义细节 (Design Tokens)

全面使用由 `rules/design-guide` 规定的准则：

- **基础色**：利用了与 iOS 极度契合的 `#F2F2F7`。
- **Slack 专属绿**：在 Contacts 用户添加按钮利用 `#2BAC76` 提供视觉联想定位。
- **键盘灰度色阶**：底盘采用 `#d1d5db`，功能灰键 (`Backspace/Shift/Next/123`) 统领 `#B3B8C2`，常规键则为纯白底黑字。

### 2. 组件状态定义

- **发送按钮态 (Button Status)**：当前处于非激活状态，因为还未完整解析有效的 Email 文本。
- **输入框交互态**：展示了一只闪烁光标停留于内容最前端（即处于空白阶段却受到 Focus），故下方唤起键盘。
- **软键盘态**：特殊变种，首行为小写 (非默认的大写锁定)，证明该字段具有 `autocapitalize="none"` 及 `keyboardType="email-address"` 属性映射特征。
